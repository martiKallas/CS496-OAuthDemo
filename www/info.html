What just happened?

When you told Google that this web app had permission to access your general info, Google sent an access code back to the web app. That access code was then exchanged for an access token. This token authorizes the bearer to access your basic profile information. This app sent that token to Google, and Google responded with the information disaplyed above as well as some additional content.

What is the state variabel?

The state variable was randomly generated by the web app when redirecting you to Google's authorization page. Google sent the same state variable back with the access code to make sure there was no interference in the hand-off of the access code.
